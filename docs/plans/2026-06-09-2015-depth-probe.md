> 状态：研究计划

> 目标：在不破坏 v1.3 正式导出逻辑的前提下，尝试突破当前 2017-09-03 左右的动态流边界，最终尝试找到并重构 2015 年的说说。

## 目标与非目标

### 目标

1. 新增独立研究路径，避免影响当前可用的 `oneclick` 正式导出流程。
2. 验证 QQ 动态流接口 `feeds2_html_pav_all` 的深度边界来自代码策略、接口参数，还是服务端真实限制。
3. 优先探索 `begin_time/end_time` 时间窗口、`offset/count` 深探、空页容忍和低并发窗口 fan-out。
4. 用可审计日志记录实验结果，确认是否能拿到早于 `2017-09-03` 的互动记录。
5. 如果能拿到 2015 年互动记录，再尝试复用现有重建逻辑构造 2015 年疑似已删除说说。

### 非目标

1. 不在第一阶段改动正式导出路径。
2. 不高并发横扫 QQ 接口。
3. 不绕过登录、风控或平台权限。
4. 不承诺完整恢复 2015 年所有说说。
5. 不把实验结果直接写入发布版，除非实验数据证明策略稳定且风险可控。

## 当前已知事实

### 当前 oneclick 动态流策略

`oneclick` v1.3 使用 `feeds2_html_pav_all` 抓动态流：

```text
begin_time=0
end_time=0
offset=<page * 100>
count=100
scope=1
format=jsonp
g_tk=<GTK>
```

当前翻页规则：

- `pageSize=100`
- `maxPages=200`
- 首页读取 `total_number`
- `total_number > 0` 时按 offset 扫到覆盖总数
- 未知总数时连续 3 个空页停止
- 中间空页会被容忍

该策略已经比 v1.1 的“连续空页停止”更稳，但它仍然信任首页 `total_number`。如果服务端给出的 `total_number` 本身被裁剪，当前代码不会主动探索边界外 offset。

### 原主程序的二分探查

主程序 `internal/infrastructure/qzone_api/qzone_api.go` 中存在 `getActivityCount`：

- 对 `offset` 做二分。
- 用 `GetActivities(offset, 100)` 是否返回活动判断边界。
- 假设 offset 空间近似单调。

风险：

- 动态流可能稀疏，中间空页不代表到底。
- 主程序旧的 `pkg/utils.ProcessOldHTML` 只提取第一个 `html:'...'` 段，会漏掉同页多个 feed。
- 因此不能直接把原主程序二分当成更可靠方案。

### 潜在突破口

`feeds2_html_pav_all` URL 中有 `begin_time/end_time`，当前固定为 `0/0`。本地没有证据证明它有效，也没有证据证明它无效。

这是最值得先验证的参数。

## 核心假设

### H1：`begin_time/end_time` 是有效时间窗口

如果成立：

- 按月或按季度扫描可以避开全量 `total_number` 裁剪。
- 2015 年窗口可能直接返回该时间段互动。
- 正式重建可从“全量深翻”转为“时间窗口合并”。

验证方式：

- 固定 `offset=0,count=100`。
- 分别请求 `2017-09`、`2017-08`、`2016-01`、`2015-01`。
- 比较原始响应大小、`total_number`、解析活动数、首尾 `timeText`。

### H2：`total_number` 是裁剪值，深 offset 外仍有数据

如果成立：

- 当前 v1.3 可能停得太早。
- 指数 offset 探查可能发现 `total_number` 之外的非空页。

验证方式：

- `begin_time=0,end_time=0`。
- 探测 offset：`0,100,200,400,800,1600,3200,6400,12800...`。
- 记录每个 offset 的响应大小、解析条数、是否有 `html:'`、是否有 `f-single`。

### H3：服务端真实边界在 2017 年附近

如果成立：

- 2015 年窗口返回空或错误。
- 深 offset 返回空、错误页或结构不含动态。
- 更激进的抓取不会带来有效增益。

验证方式：

- 多次低频探测 2017-08、2016、2015。
- 如果稳定为空，并且 2017-09 稳定有内容，则将边界视为服务端限制。

### H4：解析器漏掉更早年份不同结构的 HTML

如果成立：

- 原始响应可能非空，但当前 `parseActivities` 解析不到活动。
- 需要先扩展解析器，而不是继续加深 offset。

验证方式：

- 日志同时保存原始响应截断、`processOldHTML` 后截断、解析活动数。
- 对“原始响应大但解析为 0”的页面做人工样本分析。

## 研究实现原则

1. 新增实验模式，不改变正式导出默认行为。
2. 所有实验请求必须限速。
3. 默认并发不超过 2。

6. 出现登录态异常、连续 5xx、模板缺失或响应异常时停止实验。

## 实验阶段

### 阶段 0：建立基线

目的：确认当前 v1.3 默认策略的真实下界。

记录：

- 动态流总活动数
- 最早 `timeText`
- 最早可解析时间
- 最早活动内容摘要
- 最远 offset
- 空页数量
- 请求失败数量

验收：

- 生成 `probe_baseline_<timestamp>.jsonl`
- 明确当前下界是否仍是 `2017-09-03`

### 阶段 1：验证时间窗口参数

目的：判断 `begin_time/end_time` 是否生效。

窗口：

- `2017-09`
- `2017-08`
- `2016-01`
- `2015-01`
- `2015-06`
- `2015-12`

每个窗口只请求：

```text
offset=0
count=100
```

记录：

- `window_start`
- `window_end`
- `offset`
- `body_size`
- `has_total_number`
- `total_number`
- `has_html_segment`
- `has_f_single`
- `parsed_count`
- `min_time_text`
- `max_time_text`

判定：

- 如果不同窗口返回内容明显不同，时间窗口大概率有效。
- 如果 2015 窗口返回可解析活动，立即进入阶段 4。
- 如果所有窗口都与 `0/0` 结果相同，时间窗口可能无效。

### 阶段 2：指数 offset 深探

目的：判断 `total_number` 外是否还有数据。

探测 offset：

```text
0
100
200
400
800
1600
3200
6400
12800
20000
```

策略：

- 单线程。
- 每次请求间隔至少 500ms。
- 连续 3 次异常停止。

判定：

- 如果高 offset 有数据而中间有空页，说明当前扫描策略仍可能漏。
- 如果高 offset 稳定为空或错误，说明服务端边界概率更高。

### 阶段 3：低并发月份 fan-out

前提：阶段 1 证明时间窗口有效。

目标窗口：

- 从 `2017-09` 向前逐月探测到 `2015-01`。

并发：

- 默认 2。
- 最高 3。

每个窗口策略：

1. 请求 `offset=0`。
2. 如果有数据，继续请求 `offset=100`。
3. 若仍有数据，再做指数探测。
4. 单窗口最多请求 20 页。

停止条件：

- 同一年连续 6 个月完全无数据。
- 出现登录态异常。
- 连续 3 个窗口返回异常模板。

### 阶段 4：2015 年说说重构实验

前提：已经抓到 2015 年可解析互动。

步骤：

1. 将 2015 年互动记录按现有 `reconstructMoments` 聚合。
2. 单独输出 `reconstructed_2015_preview.html`。
3. 不与正式全量 HTML 混排。
4. 对每条重构结果标记来源窗口、互动数量、可信度。

可信度建议：

- 高：同一内容有多条评论/点赞/图片。
- 中：单条互动但有明确正文和时间。
- 低：正文为空或只有评论者痕迹。

## 风险控制

### 限速

- 阶段 1：单线程，500ms 到 1000ms 间隔。
- 阶段 2：单线程，800ms 到 1500ms 间隔。
- 阶段 3：并发 2，窗口内串行。

### 熔断

任一条件触发则停止实验：

- 连续 3 次 HTTP 5xx。
- 连续 2 次疑似登录失效。
- 响应体连续缺失 `html:'` 和 `total_number`。
- 请求耗时持续超过 30 秒。

### 隐私

- 实验日志不要记录完整 cookie。
- `g_tk` 必须脱敏。
- 活动正文可以只记录摘要或哈希，完整内容只进入本地预览 HTML。

## 第二轮真实探测结果（2026-06-09 18:37）

日志文件：

```text
oneclick/probe_offset_range_20260609_183709.jsonl
```

命令输出摘要：

```text
offset=3300..6300 step=100
31 个请求全部 status=error
错误类型全部为 HTTP 501
```

分析结论：

1. `3300..6300` 并不是普通空页，而是服务端返回 HTTP 501。
2. 每个 offset 原版本重试 3 次后仍为 501，说明不是偶发网络错误。
3. 这提示 offset 空间可能不是线性可扫，或者接口对部分 offset 形状/区间有服务端限制。

## 第六步工具升级：活动级捕获与年份预览

仅有 request 级 JSONL 摘要时，只能回答“某个 offset 是否解析到活动、最早时间是什么”，不能在发现 2015 活动后直接重建预览。因此新增两个隐藏实验能力：

```bash
./QzoneExport-macOS --probe-offset-list --capture-activities
./QzoneExport-macOS --probe-offset-range --capture-activities --offset-start 3000 --offset-end 5000 --offset-step 100
./QzoneExport-macOS --probe-depth --capture-activities
```

`--capture-activities` 会在每条 request 记录后追加同页的 activity 明细，字段包括：

```text
strategy, window, offset, index, sender_qq, sender_name, receiver_qq,
content, time_text, timestamp_unix, image_urls, activity_type
```

这些记录不包含 cookie、p_skey、g_tk 等登录凭证，但会包含空间内容摘要和图片 URL，应按隐私日志保管。

新增离线预览命令：

```bash
./QzoneExport-macOS --preview-year 2015
./QzoneExport-macOS --preview-year 2015 probe_offset_list_YYYYMMDD_HHMMSS.jsonl
```

行为：

- 默认读取同目录最新的 probe JSONL。
- 只读取本地 JSONL，不登录、不扫码、不请求 QQ 接口、不下载图片。
- 按 `timestamp_unix` 过滤目标年份。
- 对重复 activity 做去重，避免同一页被 health-check、窗口探查、offset 探查重复记录后放大互动数。
- 复用现有 `reconstructMoments -> reconAsDeletedMsgs -> buildHTML` 路径生成：

```text
reconstructed_2015_preview.html
```

限制：

- 如果探查日志没有 `type:"activity"` 明细，`--preview-year` 会提示需要用 `--capture-activities` 重新探查。
- 预览页不下载图片，只把探查日志里的图片 URL 写入 HTML；是否能显示取决于 QQ 图片直链当时是否仍可访问。

## 第七步工具升级：HTTP 501/429 自动冷却

2026-06-10 08:43 的最新探查日志：

```text
oneclick/probe_offset_list_20260610_084323.jsonl
```

结果：

```text
health-check offset=0 status=error error=HTTP 501
请求数=1 错误数=1 解析活动数=0
```

这说明接口进入短期快速拒绝状态，继续请求只会消耗机会。因此新增 probe 自动冷却：

```text
第 1 次 HTTP 501/429：冷却 5 分钟后重试同一个 offset
第 2 次 HTTP 501/429：冷却 10 分钟后重试同一个 offset
第 3 次 HTTP 501/429：冷却 20 分钟后重试同一个 offset
第 4 次 HTTP 501/429：冷却 30 分钟后重试同一个 offset
```

行为：

- 只作用于隐藏 probe 模式，不影响正式导出。
- 健康检查 `offset=0` 也会自动冷却并重试。
- 某次请求恢复成功后，退避序列重置，下次再遇到 HTTP 501/429 重新从 5 分钟开始。
- 如果四次冷却后仍失败，回到原有错误处理逻辑。
- 可用 `--no-auto-cooldown` 显式关闭。

下一轮建议命令：

```bash

./QzoneExport-macOS --analyze-probe
./QzoneExport-macOS --preview-year 2015
```

若需要分行执行，必须使用反斜杠续行，不能直接换行：

