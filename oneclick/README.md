# 一键导出 QQ 空间说说 · One-Click Qzone Export

把你 QQ 空间里**现存的全部说说**（含图片、评论）一键导出成一个**离线 HTML 网页**，
原图高清、可搜索、点击看大图，整个网页自包含（图片内嵌），断网也能看、可随意分享。

> **v1.1 新增：尝试重建“已删除”的说说。** 工具会额外扫描你的“动态流”，把当年**被点赞 /
> 评论 / 浏览过、但现在已从说说列表消失**的说说尽量还原出来，在网页里用 🕯「重建 ·
> 疑似已删除」标记，和现存说说混排展示。同时会在 exe 同目录生成一份**非常详细的日志**
> （`QzoneExport_log_<时间>.txt`），便于出问题时排查。

> **v1.3 增强：图片下载更稳。** 图片下载会自动尝试备用地址，默认 16 路并发，并对短暂断网 /
> 切换热点等情况做更耐心的重试；有效的 `cookies.json` 会继续复用，尽量避免重复扫码。

> 面向普通用户：**下载对应平台的可执行文件，双击 → 扫码 → 自动生成并打开网页。电脑上不需要安装任何东西。**

## 下载（推荐）

前往 [Releases](../../releases) 下载对应平台的文件：

| 平台 | 文件 | 用法 |
| --- | --- | --- |
| Windows | `QzoneExport.exe` | 双击运行。首次若弹“已保护你的电脑”→ “更多信息” → “仍要运行”。 |
| macOS（Intel/Apple Silicon 通用） | `QzoneExport-macOS` | 双击运行。首次若提示“无法验证开发者”→ 右键图标 → 打开 → 再点“打开”。 |

运行后：屏幕弹出二维码 → **手机 QQ 扫码并确认** → 自动抓取、下载图片 → **自动用浏览器打开**生成的 `QQ空间_你的QQ号.html`。

如需调整图片下载并发，可在命令行设置环境变量：

```bash
QZONE_DOWNLOAD_CONCURRENCY=32 ./QzoneExport-macOS
```

默认并发为 16，上限为 64。

## 它做了什么 / 为什么需要它

QQ 空间官方没有“导出说说”的功能。本工具通过你**自己登录**后的官方接口
（`emotion_cgi_msglist_v6`）翻页拉取你**现存**的说说，并：

- 自动去掉缩略图的尺寸限制参数，下载**原图**画质；
- 带 `Referer` 下载图片并 **base64 内嵌**进 HTML，避免 QQ 图床链接过期 / 防盗链导致图片丢失；
- 生成带**搜索**和**点击看大图（灯箱）**的单文件网页。

### 重建“已删除”的说说（v1.1）

抓完现存说说后，工具会再扫一遍你的**动态流**（`feeds2_html_pav_all`）。当年别人对你说说的
**点赞 / 评论 / 浏览**会在动态流里留下痕迹，即使原说说已被删除，这些痕迹有时仍残留。工具据此
反推出疑似已删除的说说，去掉与现存说说重复的部分后，用 🕯 标记混排进同一个网页。

> ⚠️ **重要边界**：这只能找回“**当年有人互动、且动态流仍残留痕迹**”的那一部分。纯文字、没人
> 互动、或腾讯服务器已彻底清除的说说，**任何工具都无法找回**。重建出的内容可能不完整（正文常被
> 截断、评论原文无法找回，只能显示评论者）。能重建多少因账号而异，可能为 0。

## 出问题怎么办（日志）

从 v1.1 起，每次运行都会在 **exe 同目录**生成 `QzoneExport_log_<时间>.txt`，记录每一步、每个
网络请求（登录凭证 `g_tk` 已脱敏、cookie 只记名字不记值）、每页抓到多少条、解析与重建明细、
以及所有错误。**如果重建不出说说或报错，请把这个日志文件发给开发者**，里面有定位问题所需的全部信息。

## 隐私与安全

- 本工具**只连接 QQ 官方服务器**，不会把你的任何数据上传到作者或第三方（源码可审计，全程只有 GET 请求、无任何上报）。
- 登录后会在同目录生成 `cookies.json`（含你的登录凭证，用于下次免扫码）。
  **用完建议删除，切勿发给任何人。**
- 生成的 HTML 含你的说说与图片，请自行妥善保管。

## 进阶 / 研究功能（v1.6，隐藏入口）

普通用户直接双击运行即可，无需关心这些。以下入口面向想**尽量找回更早回忆**的进阶用户，
均不影响默认导出流程，且抓取结果缓存在本地、可断点续传：

```bash
# 诊断：看账号到底有多少说说 / 相册 / 日志
./QzoneExport --diagnose

# 一键跨数据源找回：相册照片 + 留言板 + 日志（这些独立于说说存储，说说删了它们仍在）
./QzoneExport --recover-all

# 单独抓相册（照片带原图与上传时间，下载内嵌成网页）
./QzoneExport --harvest-photos

# 动态流深度抓取（缓存优先、慢速、断点续传），用于重建更多已删除说说
./QzoneExport --harvest --offset-end 20000 --probe-delay-ms 5000
# 纯离线分析缓存：出「重建边界报告」并生成全量重建网页（加 --download-images 可下载内嵌配图）
./QzoneExport --analyze-cache --download-images
```

> 说明：`begin_time/end_time` 时间窗口经实验证明被服务端忽略，动态流只能靠 `offset` 翻页，
> 且 offset 空间稀疏非单调；当「网格无缺口 + 尾部连续空页足够」时即可判定可重建的真实最早时间。
> 详细研究记录见 `docs/plans/`。

## 从源码编译（开发者）

纯 Go 标准库，无第三方依赖：

```bash
cd oneclick
# 当前平台
go build -o QzoneExport .
# 交叉编译 Windows
GOOS=windows GOARCH=amd64 go build -ldflags="-s -w" -o QzoneExport.exe .
# 交叉编译 macOS 通用二进制
GOOS=darwin GOARCH=arm64 go build -o qe-arm64 . && GOOS=darwin GOARCH=amd64 go build -o qe-amd64 . && lipo -create -output QzoneExport-macOS qe-arm64 qe-amd64
```

## 备选：Python 脚本（需自备 Python 3）

`qzone_export.py` 是同等功能的跨平台 Python 版；`run.bat` 是 Windows 下的启动器；
`patch_lightbox.py` 可把旧版导出的 HTML 修复成“点击看大图”的灯箱版。

```bash
python3 qzone_export.py
```

## 更新日志

### v1.6
- 🗂 **动态流本地缓存 + 慢速抓取**：新增隐藏入口 `--harvest`（缓存优先、断点续传、被风控自动冷却，
  原始响应落盘 `feed_cache/`）和 `--analyze-cache`（纯离线解析→去重→**边界报告**→全量重建 HTML）。
  抓一次存本地，之后调试/分析/重建零网络，不再暴力重复请求。
- 🔭 **真正判定重建边界**：当「offset 网格无缺口 + 尾部连续空页 ≥ 20」时，给出动态流可重建的
  真实最早时间。实测把已知最远从 2017-09-03 推进并确证到 **2017-07-25**。
- 🧰 **跨数据源回忆找回**（`recover.go`，复用零依赖架构）：
  - `--diagnose`：拉 `main_page_cgi` 计数（说说 / 相册 / 日志总数），判断 2017 是删除边界还是漏抓。
  - `--harvest-photos`：相册 → 照片（`fcg_list_album_v3` + `cgi_list_photo`，带 `uploadtime` / 描述 /
    原图），下载内嵌生成 `相册回忆_<QQ>.html`。照片独立于说说存储，说说删了照片仍在。
  - `--harvest-board`：留言板（`get_msgb`）；`--harvest-blogs`：日志（`blognew/get_abs`）。
  - `--resolve-tid <tid>`：用 `emotion_cgi_msgdetail_v6` 还原单条说说全文。
  - `--recover-all`：一键依次跑诊断 + 相册 + 留言板 + 日志。
- 🖼 **重建说说可内嵌图片**：`--analyze-cache --download-images` 会下载重建说说的配图并 base64
  内嵌进 HTML，防止 QQ 图床链接日后失效。
- 🐛 **修复相册图片 0/0 下载**：图片直链（`ExtraImgs`）此前仅在「已删除」说说时才被采集，导致相册
  照片整批漏下；改为只要有直链就采集，与删除状态无关。
- ✅ 新增缓存读写 / 命中跳过 / 边界报告（连续空页、缺口检测）/ 相册解析 / JSONP 剥离 / 上传时间
  解析等回归测试。
- ℹ️ `begin_time/end_time` 经实验证明被服务端忽略；动态流 offset 空间稀疏且非单调，详见
  `docs/plans/`。
